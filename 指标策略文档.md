# BTC 交易系统指标策略文档 (最终版)

## 一、使用的技术指标

| 指标 | 用途 |
|------|------|
| RSI (14) | 相对强弱指标，判断超买超卖 |
| MACD | 移动平均收敛/发散，判断趋势方向 |
| ADX | 平均趋向指数，判断趋势强度 |
| 布林带 (BB) | 支撑阻力位 |
| ATR | 平均真实波幅，计算止损幅度 |
| MA20 | 20日移动平均，判断价格位置和趋势 |

---

## 二、市场趋势判断

基于 **MA20斜率** 判断市场趋势：

| 趋势 | 判断条件 | 交易方向 |
|------|----------|----------|
| 上涨趋势 (up) | MA20 > 24小时前MA20 × 1.01 | 只做多 |
| 下跌趋势 (down) | MA20 < 24小时前MA20 × 0.99 | 只做空 |
| 震荡市场 (neutral) | 其他情况 | **不交易** |

> **核心改进**：只在趋势明显的市场中交易，过滤震荡市的假信号

## 五、止损止盈策略

### 1. 止损计算

**做多：**
```
止损 = 布林下轨 - 1.5 × ATR
```

**做空：**
```
止损 = 布林上轨 + 1.5 × ATR
```

### 2. 移动止损

- 当价格向有利方向移动超过 **1×ATR** 时
- 止损移动至 **开仓价（保本）**

### 3. 分批止盈

| 档位 | 盈亏比 | 说明 |
|------|--------|------|
| 第一档 | 1.5倍 | 触发后止损移至保本，继续持有 |
| 第二档 | 3.0倍 | 全部平仓 |

### 4. 平仓条件

1. 第二档止盈触发
2. 止损触发（含移动止损）
3. 超时（24小时）

## 六、仓位管理

```
单笔仓位 = 账户余额 × 风险比例 / 止损幅度%
风险比例 = 2%（单笔最大风险）
```

- 最小仓位：5 USDT
- 最大仓位：账户余额的 50%

## 八、回测结果

### 数据范围

- 数据来源: OKX API + CCXT补充
- 数据库: data/btc_data.db (表: kline_1h)
- 时间范围: 2025-01-21 ~ 2025-12-21 (约11个月)
- 数据量: 8000根1小时K线

### 不同时间区间回测结果

| 时间区间 | 天数 | 收益率 | 交易次数 | 胜率 |
|---------|------|--------|----------|------|
| 2025-06-23 ~ 12-21 | 180天 | **+121.24%** | 56 | 39.29% |
| 2025-04-24 ~ 12-21 | 240天 | +19.99% | 70 | 37.14% |
| 2025-01-21 ~ 12-21 | 365天 | -85.98% | 104 | 37.50% |

### 关键发现

1. **1-4月数据导致亏损**: 早期数据（1-4月）策略表现差，可能原因:
   - 2025年初BTC价格波动特性不同
   - 需要更多数据验证

2. **持仓时间影响大**:
   - 8小时超时: 多数盈利单被强制平仓
   - 24小时超时: 效果显著改善

3. **趋势过滤有效**: 只在趋势明确时交易，过滤了大量假信号

## 十、策略核心逻辑

```
1. 判断趋势：基于MA20斜率
   - 上涨 → 只做多
   - 下跌 → 只做空
   - 震荡 → 不交易

2. 符合趋势方向时，使用技术指标信号入场

3. 移动止损 + 分批止盈保护利润
```
