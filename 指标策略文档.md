# BTC 交易系统指标策略文档 (v2 - 加仓版)

## 一、核心策略改进

### v2 vs v1 主要区别

| 功能 | v1 | v2 |
|------|-----|-----|
| 同向信号 | 忽略 | **加仓** |
| 反向信号 | 忽略 | **平仓** |
| 每月提取收益 | 可选 | **启用** |

### 核心逻辑

```
1. 判断趋势：基于MA20斜率
   - 上涨 → 只做多
   - 下跌 → 只做空
   - 震荡 → 不交易

2. 符合趋势方向时，使用技术指标信号入场

3. 持仓期间：
   - 同向信号 → 加仓（最多加仓3次）
   - 反向信号 → 平仓（不会让亏损无限扩大）
   - 止盈/止损 → 正常平仓
   - 超时 → 平仓

4. 每月提取收益：余额超过100U时提取
```

## 一、使用的技术指标

| 指标 | 用途 |
|------|------|
| RSI (14) | 相对强弱指标，判断超买超卖 |
| MACD | 移动平均收敛/发散，判断趋势方向 |
| ADX | 平均趋向指数，判断趋势强度 |
| 布林带 (BB) | 支撑阻力位 |
| ATR | 平均真实波幅，计算止损幅度 |
| MA20 | 20日移动平均，判断价格位置和趋势 |

---

## 二、市场趋势判断

基于 **MA20斜率** 判断市场趋势：

| 趋势 | 判断条件 | 交易方向 |
|------|----------|----------|
| 上涨趋势 (up) | MA20 > 24小时前MA20 × 1.01 | 只做多 |
| 下跌趋势 (down) | MA20 < 24小时前MA20 × 0.99 | 只做空 |
| 震荡市场 (neutral) | 其他情况 | **不交易** |

> **核心改进**：只在趋势明显的市场中交易，过滤震荡市的假信号

## 五、止损止盈策略

### 1. 止损计算

**做多：**
```
止损 = 布林下轨 - 2.0 × ATR
```

**做空：**
```
止损 = 布林上轨 + 2.0 × ATR
```

### 2. 移动止损

- 当价格向有利方向移动超过 **1×ATR** 时
- 止损移动至 **开仓价（保本）**

### 3. 分批止盈

| 档位 | 盈亏比 | 说明 |
|------|--------|------|
| 第一档 | 1.5倍 | 触发后止损移至保本，继续持有 |
| 第二档 | 3.0倍 | 全部平仓 |

### 4. 平仓条件

1. 第二档止盈触发
2. 止损触发（含移动止损）
3. 超时（**12小时**）
4. **反向信号触发** → 平仓观望

---

## 五、加仓规则 (v2新增)

### 1. 同向加仓

当持仓中出现**同向信号**时，可以加仓：
- 加仓金额：账户余额 × 2%
- 最多加仓：3次
- 风险控制：反向信号会及时平仓，不会让亏损无限扩大

### 2. 反向平仓

当持仓中出现**反向信号**时，立即平仓：
- 不会再开新仓
- 等待下次同向信号再入场

> **安全保障**：因为有反向信号保护，即使加仓后杠杆变高，也能及时平仓，不会爆仓

---

## 六、仓位管理

```
单笔仓位 = 账户余额 × 风险比例 / 止损幅度%
风险比例 = 2%（单笔最大风险）
```

- 最小仓位：5 USDT
- 最大仓位：账户余额的 50%
- 加仓：同向信号时最多加3次，每次加30%仓位

---

## 七、每月提取收益

每月末检查并提取收益：

| 参数 | 值 |
|------|-----|
| 基础余额 | 100 USDT |
| 提取条件 | 余额 > 100U |
| 提取金额 | 余额 - 100U |

**示例**：
- 月末余额 150U → 提取 50U，余额保持 100U
- 月末余额 80U → 不提取，保持 80U

| 移动止损阈值 | 1×ATR |

## 九、关键发现

1. **RSI条件放宽效果显著**: 从30/70放宽到40/60后，交易机会大增
2. **持仓时间12h最佳**: 太短频繁超时，太长风险暴露过大

4. **TP3x/SL2x最优**: 盈亏比更合理

## 十一、策略优化测试（减少假突破）

### 优化方向

#### 改进1：优化入场信号
- RSI过滤区：RSI需连续保持
- MACD确认：MACD柱线为正
- 成交量验证：成交量 > 20周期均量

#### 改进2：震荡市风控
- ADX < 25 时识别为震荡市
- 仓位减半
- 禁止加仓
- 超时缩短至4小时

### 方案A：动态切换策略 (dynamic)

根据市场状态自动切换参数：
- **强趋势** (MA20斜率>1.01或<0.99 且 ADX>30)：原始v2参数
- **震荡市** (其他情况)：仓位50%、禁止加仓、超时4h

### 方案B：温和风控 (moderate)

- 只在ADX<25时仓位减半+禁止加仓
- 保留原始入场信号

### 季度回测对比

| 版本 | Q1 | Q2 | Q3 | Q4 | 全年 |
|------|-----|-----|-----|-----|---------|
| **原始v2** | +234% | +61% | -48% | +610% | **+594%** |
| 联合改进 | -3% | +83% | +25% | +131% | -33% |
| **dynamic** | +337% | +72% | -42% | +457% | **+549%** |
| **moderate** | +273% | +73% | -48% | +585% | **+618%** |

### 关键发现

1. **dynamic动态切换**：
   - 全年 +549%，接近原始+594%（仅下降7.5%）
   - Q1提升(+234%→+337%)，Q2提升(+61%→+72%)
   - Q4从+610→+457，下降明显

2. **moderate温和风控**：
   - 全年 +618%，**超越原始v2！**
   - Q2提升(+61%→+73%)
   - Q4几乎持平(+610→+585)
   - Q3无改善(-48%)

3. **推荐方案**：
   - **moderate** 为最佳选择：全年+618%，超过原始v2
   - 在ADX<25时自动减半仓，有效控制风险但不损害趋势市收益

### 结论与建议

- **趋势市(Q1、Q4)明显**：建议使用原始v2策略
- **震荡市(Q2、Q3)明显**：可考虑联合改进
- **改进2（风控）**：对Q2有提升(+61%→+77%)，对全年影响较小，可作为折中方案
