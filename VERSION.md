## v3.3 (2026-03-02) - 当前稳定版

### 问题修复
- **严重Bug修复**：OKX API每次只返回300条K线，之前回测数据不足导致结果不准确
- 修复前：只能获取约12天数据，回测结果失真（2026年显示-68%）
- 修复后：自动分页获取完整365天数据，回测结果真实

### 技术实现
- `okx_client.py`: get_klines方法增加分页逻辑，使用`after`参数循环获取历史数据
- `main.py`: 简化数据获取逻辑，直接调用get_klines即可获取所需数量

### 回测结果
| 周期 | 收益 | 交易次数 | 胜率 | 最大回撤 | 夏普比率 |
|------|------|----------|------|----------|----------|
| 2025全年 | +1185% | 186 | 46% | 106% | 1.02 |
| 2026至今(60天) | +889% | 40 | 45% | 70% | 0.15 |

### 数据说明
- K线数据保存在本地SQLite数据库
- 首次回测自动获取并保存，后续回测直接使用本地数据
- 数据范围：2025-03-02 ~ 2026-03-02

### 部署注意
- 代理地址: `http://127.0.0.1:7897`
- 如需修改，编辑 `core/okx_client.py`

## v3.1 (2026-03-02) - 当前稳定版

### 修复
- get_klines 添加代理支持
- 修复API URL路径 (/api/v5)
- 增加超时时间

### 架构
- 分离核心逻辑与执行层
- `core/trader.py` - 核心交易逻辑
- `core/executor.py` - 执行层接口

### 逻辑
- **以V2.3为准**：回测和实盘使用相同的交易逻辑
- 不限制加仓次数
- 止损先于爆仓触发

### 新增功能
- 资金划转支持（统一账户模式）
- 修复API签名问题

### 核心特性
- 趋势判断: MA20斜率
- 技术指标: RSI/MACD/布林带/ATR/ADX
- 信号过滤: 只在趋势明显的市场中交易
- 移动止损 + 分批止盈
- 加仓策略: 同向信号加仓，反向信号平仓
- 每月提取收益
- 爆仓保护

### 运行命令

```bash
# 回测
python main.py --backtest --days 365

# 实盘
python main.py --live

# 单次分析
python main.py --once
```

## 资金管理规则

### 提取+补充机制
- 每月检查余额：
  - **余额>100U**：提取超出部分
  - **余额<100U**：从已提取资金补充回100U
- 仓位计算始终基于100U
- 最终收益 = 余额100U + 累计提取 - 初始100U
