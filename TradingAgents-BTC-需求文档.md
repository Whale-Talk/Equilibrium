# TradingAgents BTC 量化交易系统 - 项目需求文档

## 1. 项目概述

### 1.1 项目背景

- **项目名称**: TradingAgents-BTC-Trader
- **项目类型**: 基于多智能体LLM的BTC合约量化交易系统
- **核心功能**: 融合TradingAgents多代理架构 + DeepSeek AI的BTC永续合约交易
- **交易平台**: OKX (OK交易所)
- **交易模式**: 合约/杠杆交易 (永续合约)
- **初始资金**: 50 USDT
- **交易品种**: BTC-USDT-SWAP (比特币永续合约)

### 1.2 核心目标

1. 实现TradingAgents多智能体架构进行市场分析
2. 通过多代理协作提高交易决策质量
3. 保留原项目的双AI架构（策略AI + 交易AI），但用TradingAgents的框架替代
4. 建立完善的风险控制机制
5. 通过Telegram实时推送交易信号

### 1.3 系统特性

| 特性 | 说明 |
|------|------|
| 多智能体分析 | 4类分析师 + 多空辩论 + 交易决策 |
| 风控系统 | 固定止损15% + 风控团队审核 |
| 记忆系统 | 记录历史交易，AI参考学习 |
| 资产追踪 | 记录收益曲线、最大回撤 |
| 模拟交易 | DRY_RUN模式，测试无忧 |
| 实时通知 | Telegram Bot推送所有重要信息 |

### 2.2 代理说明

| 代理类型 | 职责 | 输入数据 |
|----------|------|----------|
| **技术分析师** | 使用技术指标(MACD/RSI/布林带)分析价格走势 | K线数据、技术指标 |
| **新闻分析师** | 监控宏观经济和加密货币新闻 | 新闻源API |
| **情绪分析师** | 分析社交媒体情绪 | 社交媒体数据(可选) |
| **基本面分析师** | 评估BTC市值、链上数据 | 链上数据(可选) |
| **多头研究员** | 提出做多论据，质疑做空观点 | 分析师报告 |
| **空头研究员** | 提出做空论据，质疑做多观点 | 分析师报告 |
| **交易员** | 综合所有信息，做出最终决策 | 辩论结论 |
| **风控经理** | 审核交易提案，控制风险 | 交易员提案 |

## 3. 功能模块

### 3.1 数据采集模块

#### 3.1.1 K 线数据订阅
- **数据源**: OKX WebSocket API
- **订阅频道**: `candles` (K线)
- **时间周期**: 1h (1小时)、4h (4小时)、1D (1天)
- **交易对**: BTC-USDT-SWAP

#### 3.1.2 历史数据获取
- **数据源**: OKX REST API
- **获取方式**: 启动时拉取最近 100 根 K 线
- **存储方式**: SQLite 本地数据库
- **K线周期**: 1h, 4h, 1d (多周期存储)

### 3.2 技术指标模块

| 指标 | 周期 | 说明 |
|------|------|------|
| RSI | 14 | 相对强弱指标 |
| MACD | 12,26,9 | 指数平滑异同移动平均线 |
| MA | 5,10,20,60 | 移动平均线 |
| BOLL | 20,2 | 布林带 |
| ATR | 14 | 平均真实波幅 |
| ADX | 14 | 平均趋向指数 |

### 3.3 TradingAgents 多智能体模块

#### 3.3.1 技术分析师 Agent

```python

#### 3.3.2 新闻分析师 Agent

- **数据源**: Alpha Vantage API
- **注意**: 每日API调用限制25次，需节约使用
- **策略**: 仅在关键时间点调用，周期分析时不每次调用

```

### 3.4 风险控制模块

#### 3.4.1 风险规则

- **风控策略**: 由AI风控经理决策 (见3.3.5节)
- **基础规则**:

| 规则 | 阈值 | 动作 |
|------|------|------|
| 单笔最大亏损 | 15% | 强制止损 |
| 日最大亏损 | 30% | 停止交易 |
| 杠杆限制 | 最高20倍 | 自动调整 |
| 最小交易金额 | 5 USDT | 禁止下单 |

#### 3.4.2 止盈止损

- **止损**: 固定15%
- **止盈**: 动态设置 (至少1:2盈亏比)
- **追踪止损**: 开启 (回撤10%触发)

## 6. 与原项目对比

### 6.1 原项目问题

| 问题 | 原因 |
|------|------|
| AI过于保守 | 单一天真AI，倾向于观望 |
| 交易太少 | 没有多代理辩论机制 |
| 随机性强 | 缺乏结构化分析流程 |
| 止损过小 | AI给出的止损太紧 |

### 6.2 新项目改进

| 改进点 | 方案 |
|--------|------|
| 多代理协作 | 4类分析师 + 多空辩论 + 交易员 + 风控 |
| 结构化决策 | 每个代理有明确角色和任务 |
| 风险控制 | 风控经理审核交易提案 |
| 稳定输出 | 多代理辩论确保决策质量 |

## 7. 配置说明

### 7.1 配置文件

```python
# config/config.py

class Config:
    # ===== 交易模式 =====
    DRY_RUN: bool = True  # True=模拟, False=实盘

    # ===== 交易参数 =====
    TRADING_INSTRUMENT: str = "BTC-USDT-SWAP"
    LEVERAGE: int = 10
    POSITION_SIZE: float = 10.0

    # ===== 风控参数 =====
    STOP_LOSS_RATIO: float = 0.15
    TAKE_PROFIT_RATIO: float = 0.30
    DAILY_LOSS_LIMIT: float = 0.30
    MAX_LEVERAGE: int = 20

    # ===== TradingAgents 参数 =====
    DEBATE_ROUNDS: int = 2  # 辩论轮数
    USE_SENTIMENT: bool = False  # 是否使用情绪分析
    USE_NEWS: bool = True  # 是否使用新闻分析
```
